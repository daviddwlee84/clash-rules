# uv 化、AI 规则类别与 URL 修复

## 背景结论（jsDelivr 404 原因）

`examples/clash.yaml` 用的是 `@main/dist/clash/<cat>.list`，但 `/dist/` 被 `.gitignore` 排除，`main` 上没有该目录。CI（[.github/workflows/build.yml](.github/workflows/build.yml)）是把 `dist/` 的**内容**作为 `release` 分支的根目录发布的，所以正确 URL 为：

```text
https://cdn.jsdelivr.net/gh/daviddwlee84/clash-rules@release/clash/<category>.list
```

README 与 shadowrocket.conf 已是正确写法，仅 clash.yaml 需修。

## 1. 修复 examples/clash.yaml URL

- 6 个 rule-provider 的 `url` 全部从 `@main/dist/clash/...` 改为 `@release/clash/...`，并修正文件头注释（现在写的"point at the release branch"与实际 URL 不符）。

## 2. Python 全面改用 uv script

- [scripts/build.py](scripts/build.py)：加 `#!/usr/bin/env -S uv run --script` shebang + PEP 723 inline metadata（`# /// script` 块，`requires-python = ">=3.12"`，无第三方依赖故 `dependencies = []`）。
- [Justfile](Justfile)：`build` / `check` 改为 `uv run scripts/build.py`；删除 `migrate` recipe。
- [.github/workflows/build.yml](.github/workflows/build.yml)：`actions/setup-python` 换成 `astral-sh/setup-uv`，构建步骤改为 `uv run scripts/build.py`。

## 3. 删除一次性迁移脚本

- 删除 [scripts/migrate_from_legacy.py](scripts/migrate_from_legacy.py)。
- README：删 Layout 表中对应行；Provenance 部分改为注明"迁移脚本已删除，可在 git 历史中找到"。

## 4. 新增 rules/ai.list（单文件、按厂商分区注释）

新建 `rules/ai.list`，按厂商分节、每节带注释，便于后续维护。覆盖（DOMAIN / DOMAIN-SUFFIX 为主）：

- **OpenAI / ChatGPT**：openai.com, chatgpt.com, oaistatic.com, oaiusercontent.com, sora.com
- **Anthropic / Claude**：anthropic.com, claude.ai, claude.com
- **Cursor**：cursor.com, cursor.sh, cursor-cdn.com, cursorapi.com
- **Google Gemini / AI**：gemini.google.com, aistudio.google.com, generativelanguage.googleapis.com, ai.google.dev, deepmind.com, notebooklm.google.com
- **GitHub Copilot**：githubcopilot.com, copilot.microsoft.com
- **xAI / Grok**：x.ai, grok.com
- **Perplexity**：perplexity.ai, pplx.ai
- **其他**：mistral.ai, meta.ai, poe.com（DeepSeek 属大陆直连，注释说明不收录于此）

文件头注释说明用途：AI 服务普遍按 IP 区域封锁（HK 等地区不可用），建议在客户端指向专门的 `AI` 策略组（选美/日/新节点）。

接线更新：

- `examples/clash.yaml`：新增 `ai` rule-provider + `RULE-SET,ai,AI`（放在 reject 之后、proxy 之前，注释说明可降级为 PROXY）。
- `examples/shadowrocket.conf`：新增对应 `RULE-SET,...,PROXY` 行。
- README Categories 表加一行 `ai.list`。
- (看看舊有的 rules 中是否有重複的配置 特別是 proxy.list）
